3 Phase: Matching Engine Processing
3.1
Matching wird getriggert
Scheduler oder Event (Bidding-Ende) startet Matching-Prozess.
POST /matching/orders/{id}/match
Matching-Service
3.2
Lade alle validen Bids
Nur Bids mit valid=true werden berücksichtigt (Pricing-Service bereits geprüft).
SELECT FROM carrier_bids WHERE valid=true
System
3.3
Berechne Carrier Scores
Score = 0.25×priceScore + 0.25×reliability + 0.20×capacity + 0.15×distance + 0.15×risk
computeScore() → MATCH_SCORE
Matching-Service
External Data Calls:
GET /carrier-stats → reliability  |  GET /risk-score → riskScore  |  GET /capacity → capacity
3.4
Filter & Rank Carriers
Nur Carrier mit Score ≥ 0.6 qualifizieren. Sortierung absteigend nach Score.
filter(score ≥ 0.6).sort(desc)
Matching-Service
3.5
Besten Carrier zuweisen

POST /orders/{id}/assign

↓

Key Integrations

Pricing-Service Liefert priceScore (0.0-1.0) für Matching. Matching nutzt NIEMALS raw bid_price direkt.
Matching-Engine Gewichtung: 25% Preis + 25% Reliability + 20% Capacity + 15% Distance + 15% Risk
ML Pipeline Training mit executed_price_eur → kontinuierliche Verbesserung von marketPrice-Vorhersagen
